Issues P1 — Importantes

P1
P1-E1: execucoes.processo_id é nullable
A coluna processo_id é nullable, permitindo registros orphan sem processo vinculado. Viola integridade referencial.
-- Solução: Tornar NOT NULL ALTER TABLE execucoes ALTER COLUMN processo_id SET NOT NULL;
P1
P1-E2: View canônica 'execucoes_vw' não existe
Documentação referencia execucoes_vw mas apenas view_execucoes_list existe. Frontend pode falhar se referenciar nome incorreto.
-- Opção 1: Criar alias CREATE OR REPLACE VIEW public.execucoes_vw AS SELECT * FROM view_execucoes_list;
P1
P1-E3: fn_acordo_marcar_pago tem código morta
Função testa IF EXISTS (TABLE financeiro_lancamentos) que não existe. Integração é feita via trigger.
-- Apenas documentar a função (requer aprovação) COMMENT ON FUNCTION fn_acordo_marcar_pago IS 'Nota: IF EXISTS ...';
