Causa raiz: Build script não incluiu convertTypes: true no IF node, e o rightValue foi passado como string "0" em vez de número 0.

Opção A (rápida): Editar o node IF diretamente no n8n UI — trocar rightValue: "0" para rightValue: 0 (número).

Opção B (via build script): Adicionar convertTypes: true no IF node em build-mdzap-followup-zapsign.mjs e re-deploy via PATCH na API n8n.

Após o fix: testar manualmente via "Execute Workflow" no n8n UI com leads pendentes no banco.
