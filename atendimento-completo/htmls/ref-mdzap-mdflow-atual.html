4. Envio de Texto

🔄 Fluxo Completo de Envio de Texto
Usuario digita no Textarea | [Enter] (sem Shift) | MdzapMsgThread.enviar() | +----v----+ | Limpa | texto + replyMsg | estado | +---------+ | useMdzapSendMessage.sendText() | +----v-----------+ | INSERT otimista | whatsapp_mensagens | direcao: saida | (aparece imediatamente) | status: pending | | tipo: text | | metadata.preview| +----------------+ | invalidateQueries(['mdzap','mensagens',...]) | +----v-----------+ | callWebhook | mdzap/enviar-mensagem | envelope: | | numero_id | | contato | | mensagem | | tipo: text | | mensagem_id | (ID do INSERT otimista) +----------------+ | n8n faz UPDATE no registro (wa_message_id, status: sent) | Realtime UPDATE -> UI atualiza tick
Arquivo: src/hooks/mdzap/useMdzapSendMessage.ts linhas 58-109
INSERT Otimista: A mensagem aparece IMEDIATAMENTE na thread (status: pending, tick de relogio). O webhook n8n recebe o mensagem_id e faz UPDATE (nao INSERT), evitando duplicacao. O status evolui: pending -> sent -> delivered -> read (via realtime).

🎙 MdzapAudioRecorder

15. Reply / Citacao (R1)

Desktop: Botao CornerUpLeft aparece no hover da bubble (group-hover). Click seta replyMsg.

Mobile: Swipe-to-reply. Touch swipe direita (entrada) ou esquerda (saida) ate 60px. Ao atingir 50px, dispara onReply().

Banner: Quando replyMsg esta setado, aparece banner abaixo dos SmartReplies com preview da mensagem e botao X para cancelar.

Envio: O wa_message_id da mensagem citada e enviado como reply_to_wamid no INSERT otimista e no webhook.

18. Typing Indicator

Canal: Supabase broadcast mdzap-typing:{conversaId} (private).

Envio: sendTyping() chamado a cada keystroke no textarea. Envia evento is_typing: true. Apos 2.5s sem digitar, envia is_typing: false.

Recepcao: Outros usuarios aparecem com 3 dots animados + label "Fulano esta digitando...". Cleanup apos 3s sem update.

Arquivo: src/hooks/mdzap/useMdzapTyping.ts
