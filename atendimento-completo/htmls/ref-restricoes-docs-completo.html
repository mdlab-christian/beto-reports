Restricoes — Visao Geral

Restricoes sao negativacoes (Serasa, SPC, Boa Vista, Bacen/SCR, RMC, RCC) vinculadas a um cliente ou lead. Cada restricao pode ter um processo vinculado apos distribuicao. Restricoes processaveis geram pastas de documentos e podem receber peticoes via OlivIA.

Fontes de Cadastro

1. Manual (modal NovaRestricaoModal)
2. Importacao CSV (ImportarRestricoesCSVModal)
3. Consulta Bureaus (webhook n8n)
4. Auditoria CPF automatica (use_cadastrar_restricoes)

Estados
Processavel Nao Processavel Com Processo Sem Processo
Tipos (Bureau)

serasa | spc | boa_vista | bacen | scr | rmc | rcc | outro

Derivado do campo orgao_nome via deriveBureauTipo()

Diferenca entre useRestricoes vs useClienteRestricoes

useRestricoes: Usa RPC fn_restricoes_listar. Aceita filtros combinaveis (leadId, clienteId, processoId, apenasProcessaveis, apenasSemProcesso). Usado no CRM e distribuicao.

useClienteRestricoes: Query direta supabase.from('restricoes').select('*, processos(numero_cnj)'). Retorna tipo Restricao com campos derivados (restricao_tipo via deriveBureauTipo). Usado na pagina /clientes.

ATENÇÃO: Os dois hooks retornam tipos DIFERENTES de Restricao (hooks/useRestricoes.Restricao vs types/cliente.types.Restricao).

Documentos — Sistema de 3 Tiers
Tier 1 — Central
documento_lead

Docs gerais do lead/cliente.
restricao_id = NULL
Bucket: leads-docs
Upload via DocumentosUploadCentral
Aparece em TODAS as pastas

Tier 2 — Restricao
documento_restricao

Docs especificos por restricao.
restricao_id = UUID da restricao
Bucket: leads-docs
Upload via RestricaoPastaCard ou Drag&Drop
Isolado na pasta da restricao

Tier 3 — Gerado
comprovante / procuracao / contrato

Gerado por n8n via webhook.
Bucket: comprovantes ou documentos
Gerado por useGerarContrato / useGerarDocumento
Status: pendente → gerando → disponivel/erro

Fluxograma — Upload de Documentos
UPLOAD TIER 1 (Central) Usuario arrasta/seleciona arquivo na DocumentosUploadCentral │ ├── 1. Validacao: tipo aceito? (JPG/PNG/PDF/DOCX) tamanho < 20MB? ├── 2. Selecionar categoria (dropdown): RG, CPF, CNH, Comprovante, etc. ├── 3. Upload para Supabase Storage: │ bucket: 'leads-docs' │ path: ${orgId}/leads/${leadId}/${timestamp}_${filename} │ ou: ${orgId}/clientes/${clienteId}/${timestamp}_${filename} ├── 4. INSERT processos_documentos: │ processo_id: NULL, restricao_id: NULL │ categoria: 'documento_lead', tipo: 'upload' ├── 5. invalidateQueries(['lead-docs-upload', ...]) └── 6. syncToDrive (background, fire-and-forget) webhook 'sync-storage-drive' → n8n copia para Google Drive UPLOAD TIER 2 (Pasta Restricao) Dentro do RestricaoPastaCard — drop zone interna ou botao Upload │ ├── 1. Arquivo selecionado/arrastado ├── 2. Nomes predefinidos: 'Comprovante' | 'Contrato Assinado' | 'Outro' ├── 3. Upload para Storage: │ path: ${orgId}/leads/${leadId}/restricoes/${restricaoId}/${timestamp}_${filename} │ ou: ${orgId}/clientes/${clienteId}/restricoes/${restricaoId}/${timestamp}_${filename} ├── 4. INSERT processos_documentos: │ processo_id: NULL, restricao_id: UUID │ categoria: 'documento_restricao', tipo: 'upload' ├── 5. invalidateQueries(['restricao-docs-upload', restricaoId, ...]) └── 6. syncToDrive se clienteId presente UPLOAD TIER 2 (Drag & Drop sobre Card) No componente pai (ClienteDrawerTabDocumentos / ClienteDrawerTabDocumentosLead) │ ├── 1. onDragOver no card → setDragOverId(restricaoId) ├── 2. Visual: borda verde + "Solte aqui" ├── 3. onDrop → handleDropOnCard(e, restricaoId) ├── 4. Mesmo fluxo de Storage + INSERT que o Tier 2 interno └── 5. Diferenca: nao mostra dropdown de nome (usa file.name direto) GERACAO TIER 3 useGerarContrato / useGerarDocumento │ ├── 1. Usuario clica "Gerar Documento" (GerarDocModal ou DocumentoCard) ├── 2. Escolhe tipo: contrato_empresa | contrato_orgao | contrato_midas │ ou categoria: procuracao | contrato_honorarios | peticao_inicial ├── 3. webhook n8n gera PDF ├── 4. Status: pendente → gerando → disponivel/erro └── 5. Polling via refetchInterval(3000) quando status='gerando'
