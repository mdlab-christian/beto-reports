⚠ PARCIALMENTE OPERACIONAL 3 pendências externas ao escopo

O sistema de integração bidirecional entre MdFlow e Google Drive foi implementado em sua totalidade nas etapas E1 a E6. A sincronização MdFlow → Drive (upload, criação de pastas, lixeira) está operacional. A sincronização Drive → MdFlow (push notifications via Watch API) está ativa com renovação automática diária. As 3 pendências remanescentes — E7 (auto-pasta MdZap), BUG-01 (N8N-REL-CONSULTA) e BUG-02 (documentos WhatsApp) — estão fora do escopo desta implementação Drive e aguardam reconstrução da página MdZap ou ciclos separados de correção.

🔧

Etapas de Implementação (E1 — E6)

E1 Corrigir WH-SYNC-STORAGE-DRIVE ✅ Concluído
  • Workflow n8n pré-existente que sincroniza uploads do Supabase Storage para o Google Drive
  • Problema identificado: credential OAuth2 expirado + código usava https.request diretamente
  • Fix: nó reescrito para usar this.helpers.httpRequest() — gera access_token via refresh_token inline, sem depender de credential salva
  • Fix: campo corrigido de drive_file_id → google_drive_id no PATCH para processos_documentos
E2 Estrutura Hierárquica ao Criar Pasta de Cliente ✅ Concluído
  • Ao criar cliente, gera estrutura completa de pastas no Drive automaticamente via n8n
  • Pasta raiz: {Nome Cliente}/
  • Subpastas criadas: Documentos Pessoais, Restrições, Processos, Lixeira
  • Removido sufixo — {id_8char} que poluía os nomes — agora usa nome direto do cliente
  • UPDATE em clientes.google_drive_folder_id após criação

Migration SQL Aplicada (E5)

-- processos_documentos: novos campos Drive ALTER TABLE processos_documentos ADD COLUMN IF NOT EXISTS origem TEXT CHECK (origem IN ('upload_manual','whatsapp_chat','gerado_sistema','drive_sync')), ADD COLUMN IF NOT EXISTS google_drive_id TEXT, ADD COLUMN IF NOT EXISTS drive_deleted_at TIMESTAMPTZ; -- restricoes: pasta no Drive ALTER TABLE restricoes ADD COLUMN IF NOT EXISTS drive_pasta_id TEXT; CREATE INDEX IF NOT EXISTS idx_restricoes_drive_pasta ON restricoes(organizacao_id, drive_pasta_id) WHERE deleted_at IS NULL AND drive_pasta_id IS NOT NULL; -- drive_watches: controle de canais Watch API (nova tabela) CREATE TABLE IF NOT EXISTS drive_watches ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, organizacao_id UUID NOT NULL REFERENCES organizacoes(id) ON DELETE CASCADE, resource_id TEXT NOT NULL, channel_id TEXT NOT NULL, expiration TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now() NOT NULL, updated_at TIMESTAMPTZ DEFAULT now() NOT NULL ); CREATE INDEX ON drive_watches(organizacao_id); ALTER TABLE drive_watches ENABLE ROW LEVEL SECURITY; CREATE POLICY "tenant_isolation" ON drive_watches USING (organizacao_id = public.get_org_id()); CREATE TRIGGER set_updated_at BEFORE UPDATE ON drive_watches FOR EACH ROW EXECUTE FUNCTION update_updated_at_column(); -- clientes já possuía google_drive_folder_id TEXT (sem alteração)
