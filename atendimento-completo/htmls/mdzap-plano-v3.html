n8n Orchestrator

✅ Ativo
Workflow ID: HLdJDx6D45EcER9Z — Recebe toda mensagem WhatsApp e decide: Julia responde, humano atende, ou skip silencioso.
Fluxo de decisão do Orchestrator
Webhook Trigger
Recebe mensagem WhatsApp (Meta Platform)
↓
Node: Buscar Contexto
Busca conversa, lead, módulo IA no Supabase
↓
Checks em ordem (skip se verdadeiro)
1. conversa.status === 'encerrada' → SKIP
2. lead.awaiting_human === true → SKIP (Julia para!)
3. lead.wa_blocked === true → SKIP
4. metadata.agente_slug === 'pausado' → SKIP (Julia para!)
5. turn_count >= 15 → handoff automático para humano
6. Doc media sem doc_reader_result → fire-and-forget Doc Reader → SKIP
↓
se skip = true
Output
Respond Skip (HTTP 200 vazio — sem resposta ao usuário)
se skip = false
Node: Chamar Claude
Monta prompt + contexto doc reader + chama Anthropic API
↓
Output Final
INSERT mensagem DB + UPDATE metadata.turn_count++ + enviar via Meta Platform API
Mecanismo 1 — Parar Julia (lead-level)
leads.awaiting_human = true

Julia para para ESTE lead em QUALQUER conversa. Setado pelo webhook assumir-conversa.

Mecanismo 2 — Parar Julia (conversa-level)
whatsapp_conversas.metadata .agente_slug = 'pausado'

Julia para apenas para ESTA conversa específica. Granularidade fina por thread.

Bugs — Status Atualizado 2026-04-06

BUG-ASSUMIR P0 — Crítico 🔄 EM CORREÇÃO AGORA
Webhooks assumir/devolver não existiam
Root cause: Frontend chama mdzap/assumir-conversa e mdzap/devolver-julia mas esses webhooks NÃO EXISTIAM no n8n. Portanto: clicar "Assumir" falhava silenciosamente → leads.awaiting_human nunca era setado → Orchestrator nunca recebia sinal para parar → Julia continuava respondendo mesmo após humano assumir.
Fix: MDZAP-ASSUMIR-CONVERSA-v1
1 Fetch conversa → pega lead_id + metadata atual
2 Fetch user nome do perfil
3 PATCH leads SET awaiting_human = true
4 PATCH conversa metadata: mode='human', assigned_user_id, assigned_user_name, adiciona evento
5 Return action: toast_refresh
Fix: MDZAP-DEVOLVER-JULIA-v1
1 Fetch conversa → lead_id
2 PATCH leads SET awaiting_human = false
3 PATCH conversa metadata: limpa assigned_*, mode='julia', adiciona evento
4 Return action: toast_refresh
BUG-MEDIA P1 — Alta ✅ Corrigido
Imagens do WhatsApp não carregavam
Root cause: Parse Message no Facebook Inbound descartava image.url.
Fix aplicado
1 Extrai URL da imagem do payload Meta
2 Graph API → download com Bearer token
3 Upload para bucket mdzap-media no Storage
4 Salva URL permanente no banco
5 Frontend: placeholder "Processando..." sem media_url + onError "Imagem expirada"
BUG-DOCS-TIER1 P2 — Média ⏳ Pendente
Aba Docs mostra só Doc Reader results
Aba Docs exibe apenas documentos analisados pelo Doc Reader (campo metadata.doc_reader_result). Falta upload Tier 1 real via DocumentosUploadCentral + hook useLeadDocs.
