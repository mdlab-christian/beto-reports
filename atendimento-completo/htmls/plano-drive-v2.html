Arquitetura Bidirecional Completa

MdFlow → Drive (upload)
Frontend usuario faz upload de arquivo INSERT processos_documentos google_drive_id = NULL (pendente) fire-and-forget → WH-SYNC-STORAGE-DRIVE ↓ WH-SYNC-STORAGE-DRIVE (n8n) verifica: clientes.google_drive_folder_id existe? NAO → cria pasta cliente no Drive cria subpastas fixas: - Documentos Pessoais/ - Restricoes/ - Lixeira/ PATCH clientes.google_drive_folder_id SIM ↓ Supabase Storage → download binario Google Drive API → upload arquivo PATCH processos_documentos google_drive_id = "1ABC...xyz" ↓ Google Drive arquivo visivel em: {Cliente}/Documentos Pessoais/ ou {Cliente}/Restricoes/{Empresa}/
Drive → MdFlow (bidirecional)
Alguem coloca arquivo no Drive (pasta de um cliente) ↓ Google Drive Watch API detecta mudanca → envia notificacao ↓ mdflow/drive-webhook-entrada (n8n) parse: resourceId, fileId, changeType IF changeType != 'file' → ignorar ↓ GET /drive/v3/files/{fileId} obtem: name, parents, mimeType, size ↓ Code: resolver entidade qual cliente pertence a esta pasta? buscar em Supabase por: clientes.google_drive_folder_id leads_restricoes.drive_pasta_id ↓ CHECK: ja existe em processos_documentos? (por google_drive_id) SIM → ignorar (evita duplicata) NAO ↓ INSERT processos_documentos google_drive_id: fileId nome: fileName origem: 'drive_sync' lead_id / restricao_id (resolvidos) ↓ Realtime Supabase → frontend badge "☁ Drive Sync" no documento
Renovacao automatica do Watch (expira em 24h)
Google Drive Watch API expira em 24 horas — precisa renovar diariamente ↓ Cron n8n (diario — 00:00) mdflow/drive-renovar-watch ↓ SELECT organizacoes WHERE google_drive_root_id IS NOT NULL ↓ Para cada org: POST /drive/v3/files/{rootId}/watch channel.id: uuid novo channel.type: "web_hook" channel.address: "https://n8n.railway.app/webhook/mdflow/drive-webhook-entrada" ↓ PATCH organizacoes drive_watch_channel_id = channelId drive_watch_expires_at = now() + 23h ↓ Respond { done: true, orgs_renovadas: N }
ℹ️
Watch scope: configurar o watch na pasta raiz do cliente (ou na pasta da org). O Drive notifica mudancas recursivamente em subpastas. Responder HTTP 200 imediatamente — o Drive para de enviar se nao receber resposta em ~30s.

NOVO

mdflow/drive-pasta-restricao (6 nodes)

Webhook (POST /webhook/mdflow/drive-pasta-restricao) Header: mdflow: MdL1501@ Body: { cliente_id, restricao_id, empresa_nome, valor } ↓ Supabase: GET clientes WHERE id = cliente_id SELECT google_drive_folder_id ↓ IF google_drive_folder_id IS NULL → Error: "Criar pasta do cliente primeiro" → Respond { success: false, error_code: "NO_CLIENT_FOLDER" } SIM ↓ HTTP: GET /drive/v3/files q: "'{clienteFolderId}' in parents AND name='Restricoes'" fields: files(id,name) ↓ IF pasta Restricoes nao existe: HTTP: POST /drive/v3/files name: "Restricoes" mimeType: application/vnd.google-apps.folder parents: [clienteFolderId] → usar novo ID SIM → usar ID existente ↓ HTTP: POST /drive/v3/files ← pasta da restricao name: "{empresa_nome} — R$ {valor_formatado}" mimeType: application/vnd.google-apps.folder parents: [restricoesFolderId] ↓ Supabase: PATCH leads_restricoes WHERE id = restricao_id SET drive_pasta_id = novaFolderId drive_pasta_url = "https://drive.google.com/drive/folders/{id}" ↓ Respond HTTP 200 { success: true, action: "noop", data: { drive_pasta_id, drive_pasta_url } }
NOVO

mdflow/drive-lixeira (5 nodes)

Webhook (POST /webhook/mdflow/drive-lixeira) Header: mdflow: MdL1501@ Body: { google_drive_id, cliente_id } ↓ Supabase: GET clientes WHERE id = cliente_id SELECT google_drive_folder_id ↓ HTTP: GET /drive/v3/files q: "'{clienteFolderId}' in parents AND name='Lixeira'" fields: files(id,name) ↓ IF pasta Lixeira nao existe: HTTP: POST /drive/v3/files name: "Lixeira" parents: [clienteFolderId] → usar novo ID SIM → usar ID existente ↓ HTTP: PATCH /drive/v3/files/{google_drive_id} addParents: lixeiraFolderId removeParents: (parent atual do arquivo) supportsAllDrives: true ↓ Supabase: PATCH processos_documentos WHERE google_drive_id = google_drive_id SET drive_deleted_at = NOW() ↓ Respond { success: true, action: "noop" }
NOVO

mdflow/drive-webhook-entrada (8 nodes)

Webhook (POST /webhook/mdflow/drive-webhook-entrada) Nao requer header mdflow (chamado pelo Google) ↓ Code: Parse notification extrai: resourceId, fileId, changeType IF changeType != 'update' → Respond 200 + STOP ↓ HTTP: GET /drive/v3/files/{fileId} fields: id,name,parents,mimeType,size,modifiedTime ↓ IF mimeType = application/vnd.google-apps.folder → Ignorar (e uma pasta, nao arquivo) ↓ Supabase: Resolver entidade SELECT id, tipo FROM ( SELECT id, 'cliente' as tipo, google_drive_folder_id as fid FROM clientes WHERE deleted_at IS NULL UNION ALL SELECT id, 'restricao' as tipo, drive_pasta_id as fid FROM leads_restricoes WHERE deleted_at IS NULL ) t WHERE t.fid = ANY(parents) ↓ Supabase: CHECK duplicata SELECT id FROM processos_documentos WHERE google_drive_id = fileId IF existe → Respond 200 + STOP ↓ Supabase: INSERT processos_documentos google_drive_id: fileId nome: fileName origem: 'drive_sync' organizacao_id: (da org associada) lead_id / restricao_id (resolvidos) ↓ Respond HTTP 200 imediatamente (Drive exige resposta em <30s)
NOVO

mdflow/drive-renovar-watch (cron 00:00 diario)

Trigger: Schedule — todos os dias 00:00 ↓ Supabase: SELECT organizacoes WHERE google_drive_root_id IS NOT NULL ↓ SplitInBatches (1 por vez) ↓ HTTP: POST /drive/v3/files/{rootId}/watch { "id": "{uuid-gerado}", "type": "web_hook", "address": "https://primary-production-e209.up.railway.app/webhook/mdflow/drive-webhook-entrada", "expiration": "{now + 23h em ms}" } ↓ Supabase: PATCH organizacoes WHERE id = org.id SET drive_watch_channel_id = response.id drive_watch_expires_at = now() + interval '23 hours' ↓ Respond { done: true }
