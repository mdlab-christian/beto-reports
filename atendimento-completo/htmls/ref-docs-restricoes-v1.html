Documentos & Restrições

Arquitetura completa das abas de Documentos e Restrições em todos os drawers do MdFlow — CRM, Cliente, Processo e Distribuição.

1. Arquitetura Geral

As abas de Documentos e Restrições aparecem em 4 contextos diferentes do MdFlow, cada um com variações de funcionalidade:

CRM (LeadDrawer) Distribuição Página Cliente Drawer Processo

Todos compartilham os mesmos componentes base (DocumentoItem, RestricaoPastaCard, DocumentosUploadCentral) e hooks (documentos_lead_hooks.ts), mas cada contexto orquestra de forma diferente.

Lógica Interna

  1. Usuário faz drop ou click → adicionarArquivos() valida tipo MIME e tamanho
  2. Arquivos aceitos entram na fila pendentes[] com categoria vazia
  3. Usuário seleciona categoria via <select> dropdown
  4. Ao clicar "Enviar" → confirmarUpload(idx) remove da fila e chama onUpload(file, categoria)
  5. Upload imediato após escolha da categoria — sem botão extra de confirmação

13. Restrições — Visão Geral

Uma restrição representa uma negativação (Serasa, SPC, Boa Vista, etc.) vinculada a um cliente/lead. Restrições processáveis viram pastas de documentos e podem gerar processos judiciais.

17. CRM — Restrições como Pastas

No CRM, restrições processáveis são renderizadas como pastas accordion na aba Documentos. Cada pasta:

  • Mostra docs Tier 2 específicos + docs Tier 1 centrais (aparecem em todas as pastas)
  • Suporta drag & drop de arquivo → upload Tier 2 direto
  • Upload interno com nomes predefinidos (Comprovante, Contrato Assinado, Outro)
  • Botão excluir restrição (soft delete via useExcluirRestricao)
  • Multi-select com bulk download/delete

20. Realtime

Um canal realtime está configurado em ClienteDrawerTabDocumentos:

supabase.channel(`processos-documentos-cliente-${clienteId}`) .on('postgres_changes', { event: '*', schema: 'public', table: 'processos_documentos', filter: `cliente_id=eq.${clienteId}`, }, () => { queryClient.invalidateQueries({ queryKey: ['lead-docs-upload'] }) queryClient.invalidateQueries({ queryKey: ['restricao-docs-upload'] }) }) .subscribe()

Cleanup obrigatório via return () => supabase.removeChannel(channel). Ativado apenas quando clienteId disponível. Garante que uploads por outro dispositivo/aba atualizam instantaneamente.
