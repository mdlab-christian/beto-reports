1. Visao Geral

78% completo

O MdZap e o modulo de atendimento WhatsApp do MdFlow. Integra a Julia (IA conversacional) com um painel humano completo para SDRs e advogados da MP Advocacia. O fluxo cobre desde a primeira mensagem do lead ate o arquivamento e distribuicao ao juridico.

Nota atual: 7.8 / 10 — core funcional e robusto, mas 3 bugs P0 impedem operacao plena.

9. Julia & Orchestrator

Orchestrator — Motor de decisao

O Orchestrator e o cerebro do MdZap. Recebe cada mensagem agrupada do message-grouper e decide o que fazer.

CondicaoAcaoResultado
awaiting_human = true SKIP Humano assumiu — Julia nao responde
agente_slug = 'pausado' SKIP Agente pausado manualmente
status = 'encerrada' SKIP Conversa ja encerrada/arquivada
wa_blocked = true SKIP Lead bloqueou ou foi bloqueado
Mensagem tipo doc/image Doc Reader Fire-and-forget para mdzap/doc-reader, aguarda resultado
_source = 'doc_reader' Responde Re-trigger com contexto do doc processado
turn_count >= 15 Handoff Muitas interacoes — passa para humano automaticamente
Mensagem de texto normal Julia Claude gera resposta como Julia e envia
Julia — Prompt e comportamento

Julia e a persona de SDR da MP Advocacia. Age como atendente simpatica, profissional, que:

  • ✓ Coleta dados do lead (nome, CPF, endereco)
  • ✓ Explica o servico de remocao de restricoes
  • ✓ Solicita documentos (RG, comprovante residencia)
  • ✓ Encaminha para auditoria quando tem CPF
  • ✓ Faz handoff para humano quando necessario

Prompt management: Configuravel via Julia Lab e Admin IA no frontend. Prompts salvos em mdzap_ai_modules.

AI Composer: Ferramentas de reescrita para o atendente humano — reescrever, completar, tornar formal, simplificar, traduzir.

10. Assumir / Devolver

Bug P0-003
Botao Assumir

Webhook: mdzap/assumir-conversa

Workflow: MDZAP-ASSUMIR-CONVERSA-v1

O que faz:

  • 1. SET assigned_user_id = user atual
  • 2. SET awaiting_human = true
  • 3. Adiciona evento na timeline
  • 4. Julia para de responder (skip condition)

Funcional

Botao Devolver

Webhook: mdzap/devolver-julia

Workflow: MDZAP-DEVOLVER-JULIA-v1

O que faz:

  • 1. SET assigned_user_id = null
  • 2. SET awaiting_human = false
  • 3. Adiciona evento na timeline
  • 4. Julia volta a responder

Bug P0-003

P0-003 — Detalhamento tecnico:
Condicao atual: Devolver so aparece quando lead_awaiting_human === true.
Problema: Apos assumir, awaiting_human pode ja estar false no carregamento seguinte (race condition ou reset por outro workflow).
Resultado: Atendente fica "travado" — nao consegue se desassignar.
Fix proposto: Mostrar Devolver quando assigned_user_id esta set (independente de awaiting_human).

15. Bugs Ativos

9 bugs

P0 — Criticos (bloqueiam operacao)

P0-001 sendLink sem INSERT otimista Blocker
Link enviado nao aparece na thread ate o n8n processar e retornar. Todos os outros tipos (texto, midia) aparecem imediatamente via INSERT otimista.
Fix: Adicionar INSERT otimista em sendLink igual ao padrao de sendText — INSERT local com status: 'pending' antes de chamar webhook.
P0-002 Mensagens pending sem cleanup/rollback Blocker
Mensagens otimistas ficam travadas em status: 'pending' para sempre se o webhook falha. Nao ha cleanup, rollback, nem indicacao visual de falha.
Fix: Timeout de 10s apos envio. Se nao recebe confirmacao, marcar como status: 'failed' com indicador visual (icone de erro + botao "tentar novamente").
P0-003 Botao Devolver ausente apos assumir Blocker
Devolver so aparece quando lead_awaiting_human === true. Apos assumir, awaiting_human pode ja estar false no carregamento seguinte. Atendente fica "travado".
Fix: Condicao do botao Devolver deve ser assigned_user_id !== null (independente de awaiting_human).

P1 — Importantes (degradam experiencia)

P1-001 Bulk archive UPDATE direto sem webhook
Selecao multipla + arquivar faz UPDATE direto no banco. Logica adicional do workflow (mover lead CRM, pausar Julia) nao e executada.
Fix: Iterar sobre conversas selecionadas e chamar mdzap/arquivar-conversa para cada uma (ou criar endpoint bulk no n8n).
P1-002 Audio gravado envia como base64
Audio gravado envia como base64 no payload JSON. Pode gerar payload enorme para audios longos.
Fix: Upload do audio para Storage mdzap-media antes de chamar webhook (igual ao fluxo de imagem/video).
P1-003 Limite 100 mensagens sem paginacao
Query busca apenas as ultimas 100 mensagens. Conversas longas perdem historico antigo sem forma de recuperar.
Fix: Implementar paginacao cursor-based com botao "Carregar mais" no topo da thread.
P1-004 Campo CPF sem validacao/mascara
Campo CPF na aba Dados aceita qualquer string. Sem mascara de formatacao e sem validacao de digitos verificadores.
Fix: Mascara ###.###.###-## + validacao algoritmo digitos verificadores antes de salvar.
P1-005 as any extensivo em queries Supabase
Schema TypeScript desatualizado forca uso de (supabase as any) em varias queries.
Fix: npx supabase gen types typescript e substituir todos os as any.
P1-006 rpc_disparar_auditoria sem type safety
RPC chamado com (supabase as any).rpc(). Nenhuma validacao de parametros em compile-time.
Fix: Regenerar types (P1-005) resolve automaticamente.

P2 — Melhorias (nice-to-have)

P2-001 Doc Reader sem prompt configuravel no Admin IA
Os tipos de prompt no Admin IA nao incluem doc_reader. Prompt configurado apenas no n8n (Code node).
Fix: Adicionar tipo doc_reader em mdzap_ai_modules + UI no Admin IA.
P2-002 Dashboard periodo selector desabilitado
Seletor de periodo mostra "em breve" permanentemente. KPIs usam apenas periodo padrao.
Fix: Implementar filtro de datas com query parametrizada no dashboard.
