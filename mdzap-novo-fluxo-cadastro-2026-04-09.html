Plano de Implementação — 5 Etapas em Ordem Obrigatória

⛔ GAP CRITICO — Pré-requisito Bloqueante
Problema: o workflow n8n Doc Reader (eVgkSfkq2Clrlzuq) extrai os dados do documento e escreve em whatsapp_mensagens.metadata.doc_reader_result — mas nunca propaga esse dado para whatsapp_conversas.metadata.pending_lead_data.

Consequência: o Lead Panel não tem de onde ler os dados pré-preenchidos pela Julia. O botão "Auditoria + Cadastro" aparece com campos vazios.

Fix: adicionar 1 nó Supabase PATCH ao final do Doc Reader, após extração bem-sucedida, escrevendo o resultado em whatsapp_conversas.metadata via jsonb_set.
ETAPA 1 — PREREQ
Fix n8n Doc Reader
Complexidade S
n8n workflow: eVgkSfkq2Clrlzuq
Adicionar nó Supabase após extração bem-sucedida
PATCH whatsapp_conversas SET metadata = jsonb_set(metadata, '{pending_lead_data}', extracted_data) WHERE id = conversa_id
Executar apenas quando doc_reader_result.status = 'success'
Nenhum bloqueador — pode executar agora
