N8N — Facebook Inbound

13 Nós
NÓ 1 Webhook GET Verify GET
Verifica o hub challenge do Meta. Executado uma única vez no setup da integração.
Trigger: GET /webhook/mdzap-facebook Verifica: hub.verify_token == 'mdzap_fb_verify_2026' → true: Respond hub.challenge (200) → false: Respond 403 Forbidden
NÓ 2 Webhook POST POST
Recebe todos os eventos da Meta. Retorna 200 imediatamente (onReceived) para evitar retentativas do Meta.
Trigger: POST /webhook/mdzap-facebook Respond: imediato (onReceived) → 200 OK Continua: processamento assíncrono
NÓ 3 Parse & Route Type BRANCH
Extrai o payload do Meta e roteia pelo tipo de evento.
payload = entry[0].changes[0].value Branch A: value.messages → processar mensagem Branch B: value.statuses → atualizar status de entrega Branch C: (else) → noop — evento sem dados relevantes
NÓ 4A Lookup Numero SQL
Para Branch A (mensagem). Resolve o phone_number_id Meta para um registro interno, carregando org, agente e token.
phone_number_id = value.metadata.phone_number_id SELECT id, organizacao_id, agente_slug, config FROM whatsapp_numeros WHERE wa_instance_id = $phone_number_id AND deleted_at IS NULL → Retorna: numero_id, organizacao_id, agente_slug, access_token
NÓ 4B Update Status SQL
Para Branch B (status_update). Atualiza o status de entrega de cada mensagem enviada pela Julia.
Para cada status em value.statuses: UPDATE whatsapp_mensagens SET status = $status WHERE wa_message_id = $wamid AND deleted_at IS NULL Status possíveis: sent → delivered → read failed (qualquer etapa)
Fim do fluxo para o Branch B.
NÓ 5 Parse Message PARSE
Normaliza todos os campos da mensagem Meta em variáveis padronizadas.
from = message.from (telefone cliente) contact_name = contacts[0].profile.name wamid = message.id (wa_message_id) timestamp = message.timestamp message_type = message.type (text|image|video|audio|document|location|sticker) text = message.text?.body (se type=text) media_id = message[type]?.id (se mídia) caption = message[type]?.caption filename = message.document?.filename mime_type = message[type]?.mime_type latitude = message.location?.latitude longitude = message.location?.longitude
NÓ 6 Upsert Conversa UPSERT
Busca conversa ativa existente ou cria uma nova. Incrementa contador de não lidas.
-- Busca conversa ativa SELECT id, nao_lidas FROM whatsapp_conversas WHERE contato_numero = $from AND numero_id = $numero_id AND deleted_at IS NULL LIMIT 1 -- Se existe: UPDATE SET ultimo_msg_at = now(), nao_lidas = nao_lidas + 1 -- Se não existe: INSERT (organizacao_id, numero_id, contato_numero, contato_nome, status='ativa', nao_lidas=1) → RETURNING id as conversa_id
NÓ 7 Branch por Tipo de Mensagem BRANCH
text | location | sticker → Nó 8 (Save Message direto) audio → Nó 9A (Download → Transcribe → Save) image | video | document → Nó 9B (Download → Save → Vision opcional)
NÓ 8 Save Message — Texto / Location / Sticker INSERT
INSERT INTO whatsapp_mensagens: organizacao_id = $organizacao_id conversa_id = $conversa_id direcao = 'entrada' tipo = $message_type remetente_tipo = 'human' status = 'received' wa_message_id = $wamid metadata = { preview: $text[0:200], from: $from, contact_name: $contact_name, timestamp: $timestamp, latitude: $latitude?, longitude: $longitude? } RETURNING id as mensagem_id
NÓ 9A Download Media HTTP
Baixa mídia da Meta e faz upload no Supabase Storage. Executado para áudio, imagem, vídeo e documento.
1. GET https://graph.facebook.com/v19.0/{media_id} Header: Authorization: Bearer {access_token} → { url, mime_type } 2. GET {url} Header: Authorization: Bearer {access_token} → download binário do arquivo 3. Upload Supabase Storage: bucket: mdzap-media path: {organizacao_id}/{conversa_id}/{wamid}.{ext} → storage_url (URL pública ou signed)
NÓ 9B Save Message — Mídia INSERT
INSERT INTO whatsapp_mensagens: organizacao_id = $organizacao_id conversa_id = $conversa_id direcao = 'entrada' tipo = $message_type remetente_tipo = 'human' status = 'received' wa_message_id = $wamid metadata = { preview: $caption || $filename || '[áudio]', media_id: $media_id, storage_url: $storage_url, mime_type: $mime_type, filename: $filename?, caption: $caption?, from: $from, contact_name: $contact_name, timestamp: $timestamp } RETURNING id as mensagem_id
NÓ 9C Transcrição de Áudio POST (só audio)
Transcreve áudio via Whisper. O texto transcrito é injetado no contexto da Julia como se fosse texto.
POST /mdzap/audio-transcribe Body: { storage_url, conversa_id, mensagem_id } → Whisper transcreve arquivo → UPDATE whatsapp_mensagens SET metadata = jsonb_set(metadata, '{transcricao}', $texto) WHERE id = $mensagem_id → transcricao usada como $text no contexto do Orchestrator
NÓ 9D Document Received POST (document/image)
Processa documentos recebidos — salva em leads_documentos e opcionalmente analisa com Vision (Claude/Gemini).
POST /mdzap/document-received Body: { storage_url, conversa_id, lead_id, mime_type, filename } → INSERT leads_documentos (Tier 1 — não validado) → Vision Analysis (opcional): Claude/Gemini detecta tipo do documento Exemplos: RG, CPF, comprovante renda, contrato → UPDATE leads_documentos SET tipo_detectado = $tipo
NÓ 10 Router SDR SWITCH
Roteia para o orquestrador correto com base no agente_slug do número que recebeu a mensagem. Ponto de extensão para SDR 2 e 3.
agente_slug = lookup_resultado.agente_slug switch (agente_slug): 'sdr_leads' → POST /webhook/mdflow/mdzap/orchestrator Body: { conversa_id, agente_slug: 'sdr_leads', mensagem_id } 'sdr_docs' → POST /webhook/mdflow/mdzap/sdr-docs-stub Body: { conversa_id } (acknowledge — implementação futura) 'sdr_acordos' → POST /webhook/mdflow/mdzap/sdr-acordos-stub Body: { conversa_id } (acknowledge — implementação futura) default → log: "agente_slug desconhecido: {slug}"

N8N — Orchestrator

11 Nós · Claude claude-sonnet-4-6
Recebe: POST /mdflow/mdzap/orchestrator com body { conversa_id, agente_slug, mensagem_id }
NÓ 1 Webhook Trigger POST
POST /mdflow/mdzap/orchestrator Header: mdflow: MdL1501@ Body: { "conversa_id": "uuid", "agente_slug": "sdr_leads", "mensagem_id": "uuid" } → responseNode ao final do fluxo (não imediato)
NÓ 2 Buscar Contexto 4x SQL
-- 1. Conversa SELECT c.id, c.contato_numero, c.status, c.metadata, c.lead_id, c.numero_id, c.organizacao_id FROM whatsapp_conversas c WHERE c.id = $conversa_id AND c.deleted_at IS NULL -- 2. Lead (se lead_id não nulo) SELECT l.id, l.nome, l.telefone, l.cpf, l.coluna_crm, l.awaiting_human, l.wa_blocked FROM leads l WHERE l.id = $lead_id -- 3. Histórico (últimas 20, cronológico reverso) SELECT direcao, tipo, remetente_tipo, metadata, created_at FROM whatsapp_mensagens WHERE conversa_id = $conversa_id AND deleted_at IS NULL ORDER BY created_at DESC LIMIT 20 -- 4. Módulo IA SELECT system_prompt, modelo, temperatura, max_tokens, config FROM mdzap_ai_modules WHERE slug = $agente_slug AND ativo = true
NÓ 3 Skip Guard IF
Se qualquer condição for verdadeira, responde 200 silenciosamente sem chamar a IA.
SKIP se: conversa.status = 'encerrada' lead.awaiting_human = true lead.wa_blocked = true modulo IS NULL (slug não encontrado no banco) última mensagem de saída < 3s atrás ← anti-loop
NÓ 4 Build Messages Array TRANSFORM
Converte histórico do banco para o formato messages[] da API Anthropic.
historico (reverso → cronológico) → messages[] direcao='entrada' → { role: 'user', content: metadata.transcricao || metadata.preview || '[mídia sem texto]' } direcao='saida' → { role: 'assistant', content: metadata.preview } Regras: ✦ Máximo 20 mensagens ✦ Remove messages com content vazio/nulo ✦ Array DEVE começar com role='user' ✦ Alternância user → assistant obrigatória
NÓ 5 Build System Prompt TEMPLATE
{modulo.system_prompt} === CONTEXTO DO CLIENTE === Nome: {lead.nome || 'não identificado'} Telefone: {conversa.contato_numero} CPF: {lead.cpf || 'não informado'} Status CRM: {lead.coluna_crm || 'novos'} Turno: {metadata.turn_count || 0} Data/hora: {ISO timestamp BRT} === INSTRUÇÕES ESPECIAIS === - Se cliente pedir humano: responda e inclua "HANDOFF_REQUESTED" - Se precisar do CPF: pergunte educadamente - Respostas curtas, WhatsApp-friendly, sem markdown pesado
NÓ 6 Chamar Claude API LLM
POST https://api.anthropic.com/v1/messages Headers: x-api-key: $env.ANTHROPIC_API_KEY anthropic-version: 2023-06-01 Content-Type: application/json Body: { "model": modulo.modelo || "claude-sonnet-4-6", "max_tokens": modulo.max_tokens || 1024, "temperature": modulo.temperatura || 0.7, "system": $system_prompt_construido, "messages": $messages_array }
NÓ 7 Parse Resposta Claude PARSE
resposta_texto = content[0].text tokens_input = usage.input_tokens tokens_output = usage.output_tokens pedido_humano = /HANDOFF_REQUESTED|falar com (uma )?pessoa|atendente|humano/i .test(resposta_texto) cpf_extraido = /\d{3}\.?\d{3}\.?\d{3}-?\d{2}/.exec(client_msg) (aplicado na última mensagem do usuário)
NÓ 8 Salvar Resposta no Banco INSERT + UPDATE
-- Insere mensagem de saída INSERT INTO whatsapp_mensagens: organizacao_id = $organizacao_id conversa_id = $conversa_id direcao = 'saida' tipo = 'text' remetente_tipo = 'nina' ← ver P5 (inconsistência com Julia) status = 'pending' ← atualizado para 'sent' após envio Meta metadata = { preview: $resposta_texto[0:200], tokens_input: $tokens_input, tokens_output: $tokens_output, agente_slug: $agente_slug } RETURNING id as mensagem_saida_id -- Atualiza metadados da conversa UPDATE whatsapp_conversas SET metadata = jsonb_set(metadata,'{turn_count}', (COALESCE((metadata->>'turn_count')::int,0)+1)::text::jsonb), metadata = jsonb_set(metadata,'{last_julia_at}', now()::text::jsonb), updated_at = now() WHERE id = $conversa_id
NÓ 9 Handoff Guard IF
Se detectou pedido de humano, marca a conversa como awaiting_human e registra evento de CRM.
SE pedido_humano = true: UPDATE leads SET awaiting_human = true WHERE id = $lead_id INSERT crm_sync_events: event_type = 'conversation.status_changed' reason = 'handoff_requested' → Continua fluxo normalmente para enviar mensagem de transição
NÓ 10 Enviar via Meta POST
POST https://primary-production-e209.up.railway.app /webhook/mdflow/mdzap/enviar-mensagem Headers: mdflow: MdL1501@ Body: { "request_id": "uuid-gerado", "contexto": "mdzap.orchestrator-reply", "organizacao_id": "$organizacao_id", "actor_user_id": null, "origem_rota": "/mdzap", "data": { "numero_id": "$numero_id", "contato": "$contato_numero", "mensagem": "$resposta_texto", "tipo": "text", "mensagem_id": "$mensagem_saida_id" ← para UPDATE otimista } }
NÓ 11 Respond 200 RESPONSE
HTTP 200 OK { "success": true, "action": "noop", "message": "resposta enviada" }

Problemas a Corrigir — F1

4 Blockers · 1 Aviso
P1 — BLOCKER
wa_numero_id hardcoded no FACEBOOK-INBOUND
Arquivo: k9377WTyVJYOnD4e
Nó: Parse Message
O numero_id está fixo no fluxo em vez de ser resolvido dinamicamente pelo phone_number_id da Meta. Com isso, qualquer número adicional (SDR 2, SDR 3) quebraria silenciosamente.
Fix: extrair metadata.phone_number_id no Nó 3 e executar SELECT dinâmico em whatsapp_numeros WHERE wa_instance_id = $phone_number_id
P1 — BLOCKER
FACEBOOK-INBOUND não chama o Orchestrator
O fluxo atual termina no Save Message sem encaminhar para nenhum orquestrador. A Julia nunca responde porque o cenário de IA não é disparado.
Fix: adicionar Nó 10 "Router SDR" após o Save Message, chamando POST /webhook/mdflow/mdzap/orchestrator com {conversa_id, agente_slug, mensagem_id}
P1 — BLOCKER
Orchestrator com agente_slug 'sdr_leads' hardcoded
O SELECT em mdzap_ai_modules usa um slug fixo em vez de ler o valor recebido no payload. Impossível expandir para múltiplos SDRs sem duplicar o workflow inteiro.
Fix: receber agente_slug no body do webhook e usar $agente_slug dinamicamente no SELECT WHERE slug = $agente_slug
P1 — BLOCKER
API key Anthropic hardcoded no Orchestrator
A chave ANTHROPIC_API_KEY está inserida literalmente no nó HTTP do Orchestrator. Risco de exposição em logs, histórico de versões do n8n e exports de workflow.
Fix: substituir valor literal por {{ $env.ANTHROPIC_API_KEY }} via n8n Environment Variables
P3 — AVISO
remetente_tipo='nina' inconsistente com nome Julia
Mensagens saída inseridas com remetente_tipo='nina', mas o agente em produção chama-se Julia (sdr_leads). Inconsistência cosmética, mas pode confundir filtros futuros na UI.
Fix: atualizar para remetente_tipo='julia' — ou decidir padronizar como 'ia' e guardar o nome do agente em metadata.agente_slug
