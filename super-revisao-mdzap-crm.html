🟡Achados P1 — Importantes
⚠️
RLS — leads, auditorias, crm_sync_events usam org_id() deprecated

O padrão MdFlow exige get_org_id(). Estas 3 tabelas core ainda usam org_id(). Funcionalmente equivalente hoje, mas risco de quebra se org_id() for removido. Fix: Migration para atualizar policies para get_org_id().

⚠️
restricoes — RLS é apenas ALL policy, sem granularidade CRUD

Tabela restricoes tem apenas restricoes_org_isolation (ALL). Deveria ter policies separadas para SELECT, INSERT, UPDATE, DELETE com regras específicas (ex: DELETE somente admin). Fix: Criar 4 policies granulares.

⚠️
crm_notas — RLS é apenas ALL policy

Mesmo problema de restricoes. Policy tenant_isolation cobre tudo mas sem granularidade. Fix: Adicionar policies CRUD.

A coluna restricoes.data_restricao é date. O padrão MdFlow exige TIMESTAMPTZ. Neste caso, porém, faz sentido manter DATE pois restrições têm apenas data (sem hora).

💡
leads tem 64 colunas — potencial de refatoração

A tabela leads cresceu para 64 colunas. Campos como wa_* (5 cols) e endereco_* (8 cols) poderiam ser normalizados em tabelas separadas ou JSONB. Não é bloqueante mas aumenta complexidade.

💡
fn_restricao_salvar não aparece na lista de RPCs

O hook useSalvarRestricao chama fn_restricao_salvar mas essa RPC não apareceu no inventário. Pode ter nome diferente ou não estar criada. Verificar no banco.
