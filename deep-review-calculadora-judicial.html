Visao do Projeto (Resumo do Joao)

O que e a Calculadora Judicial?

Sistema de calculo judicial tipo DRCalc, mas muito mais moderno, claro e poderoso. Baseado em linha do tempo com regras por periodo. Projetado para advogados que precisam calcular valores de condenacao, execucao e cumprimento de sentenca com precisao e rastreabilidade total.

Funcionalidades-chave

  • Motor baseado em LINHA DO TEMPO com marcos temporais e periodos independentes
  • Correcao monetaria por periodo: IPCA, IPCA-E, INPC, IGPM, TR, Selic (6 indices)
  • Juros em 4 modos: simples, composto, taxa fixa, Selic, taxa legal art. 406
  • Selic em 3 modos: exclusiva, com correcao separada, so por periodo
  • Eventos financeiros: pagamentos parciais, depositos judiciais, abatimentos com recalculo
  • Multas: 523 CPC (10%), astreintes com teto, multas contratuais
  • Honorarios: fixos ou % sobre 5 bases diferentes de calculo
  • Checagem automatica de consistencia (12 validacoes)
  • Modelos prontos (templates) para acelerar preenchimento
  • Camada IA futura: leitura do titulo executivo para sugerir parametros
  • Relatorio exportavel profissional (PDF/Excel) auditavel

Decisoes Arquiteturais

Q1: Calculo no frontend ou backend?
Frontend (motor JS/TS puro). O calculo roda no browser em tempo real. Backend so persiste. Motivo: UX instantaneo, sem latencia de rede a cada mudanca de parametro.
Q2: Series de indices: API BCB em tempo real ou cache?
Cache local na tabela indice_valores. Series atualizadas mensalmente via cron n8n. Cache React Query de 1h. Motivo: indices mudam 1x/mes, nao faz sentido chamar API a cada calculo.
Q3: Selic exclusiva vs separada — como modelar?
Campo selic_modo em calculo_periodos com 3 valores CHECK. O motor decide a logica baseado nesse campo. Ja implementado.
Q4: Snapshot de calculo finalizado — imutabilidade?
fn_calculo_snapshot_criar grava JSON completo (parametros + resultado + indices) com versao. Calculo finalizado nao aceita mais edicoes. Snapshot preserva a base de indices usada na data.
