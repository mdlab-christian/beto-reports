Navegação — O Problema das 2 Camadas

O MdZap tem 2 níveis de navegação que confundem:

Nível 1: Tabs Top-level

Conversas | Dashboard | Julia Lab | Templates | Gestão

Muda a tela inteira. Claro.

Nível 2: SDR Mode Tabs

Leads | Docs | Acordos

Filtra conversas por SDR mode. Confuso: “Docs” parece ser aba de documentos mas é filtro SDR.

P1 Quando o usuário clica “Docs” no topo, espera ver documentos. Em vez disso, a lista de conversas filtra por SDR mode “docs” e se não há conversas nesse mode, mostra “Nenhuma conversa ativa” com lead panel vazio. Sugestão: Renomear SDR tabs para “SDR: Leads” / “SDR: Docs” / “SDR: Acordos” ou usar ícones diferenciados.

Bug Visual Confirmável — Bureau Error Exposto

Na aba Restrições do Lead Panel, o erro do bureau é visível ao usuário final:

Consulta falhou
Todos os bureaus falharam: apifull_115
(requisição inválida: Não houve resposta da API!)

Isso confirma o BUG-01 (bureau mismatch) do code audit. O usuário vê a mensagem técnica do backend.

Fix (UX): Mensagem deveria ser “Não foi possível consultar restrições. Tente novamente.” sem expor nome do bureau.

Plano de Ação

P0 Fix Imediato

  1. Data pt-BR: Trocar formato de nascimento para dd/MM/yyyy. (~5min)
  2. Bureau error msg: Trocar mensagem técnica por amigável no CrmAuditoriaPanel. (~10min)
  3. Touch targets mobile: CSS @media (max-width: 640px) { button, [role=tab] { min-height: 44px; } } (~15min)

P1 Melhorias Urgentes

  1. Markdown rendering: Adicionar parser básico (bold/italic) nas mensagens Julia. (~30min)
  2. Contato fallback: Quando nome é vazio ou “.”, mostrar número formatado. (~10min)
  3. SDR tabs naming: Prefixar com “SDR:” ou usar ícones distintos. (~10min)
  4. Mobile thread: Implementar slide-in ou push navigation. (~2h)
  5. CPF validation blocking: Desabilitar submit quando CPF inválido. (~10min)

Veredicto Final

6.2/10 — A melhor nota das 4 etapas, o que diz mais sobre as outras etapas do que sobre esta. O frontend visual do MdZap demonstra disciplina com o Design System v5 — zero cores hardcoded, tipografia correta, console limpo. O layout desktop é funcional e as 5 tabs oferecem boa organizacao. Dashboard e Julia Lab estão acima da média.

O que puxa para baixo: data em formato americano (P0 indefensável), erro técnico do bureau exposto ao usuário, 15 touch targets violando o mínimo 44px no mobile, thread inacessível em <768px, e navegação confusa entre tabs top-level e SDR modes. O caminho para 8.0 exige: corrigir data (5min), error msg (10min), touch targets (15min), markdown (30min), mobile thread (2h). Total: ~3h.
