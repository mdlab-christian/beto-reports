Fluxo de Atendimento

11 fases
1
Fase 1 Entrada do Lead Julia
Lead chega via WhatsApp mandando o CPF na primeira mensagem e algo padrão como "tenho interesse". Julia recebe e inicia o fluxo de qualificação.
2
Fase 2 Julia apresenta o processo Julia
Julia explica o processo e pede os documentos necessários para análise do caso.
3
Fase 3 Pedido de documento de identificação Julia
Julia pede identidade ou CNH. Com esse documento: nome completo, CPF, data de nascimento, RG, sexo.
4
Fase 4 Processamento da identidade / CNH Doc Reader (Sonnet)
Subagente processa o documento recebido, extrai dados pessoais (nome, CPF, data nascimento, RG, sexo), salva como PDF nomeado, preenche aba Dados automaticamente, envia JSON para Julia.
5
Fase 5 Pedido de comprovante de residência Julia
Logo após recebimento da identidade/CNH, Julia solicita o comprovante de residência.
6
Fase 6 Processamento do comprovante + fluxo de terceiro Doc Reader + Julia
Caso A (nome do próprio): Doc Reader extrai endereço completo, preenche, salva PDF.

Caso B (nome diferente = terceiro): Doc Reader não salva → passa para Julia → Julia explica que precisa de declaração com foto obrigatória do documento de identificação: "eu (nome) portador do CPF (cpf), declaro que (nome do lead) portador do CPF (cpf do lead), reside nesse comprovante de residência" + foto do doc de identificação abaixo da declaração.
7
Fase 7 Cadastro automático do lead Sistema
Com os dados dos documentos (nome, CPF, data nasc, RG, sexo, endereço completo) + número WhatsApp já conhecido, sistema cadastra o lead. Lead vai para coluna em_atendimento no CRM.
Bug ativo: última implementação do cadastro automático gerou valores errados nas restrições — corrigir no E-17.
8
Fase 8 Auditoria e restrições Humano
Atendente assume, faz auditoria do CPF (botão Nova Auditoria), resultado abre em nova guia imediatamente. Cadastra as restrições no CRM. Cria pastas do lead.
9
Fase 9 Lead em atendimento — aguardando profissão Julia
Lead só pode mover de coluna quando tiver profissão preenchida. Julia pede a CTPS (carteira de trabalho).
10
Fase 10 Processamento da CTPS Doc Reader (Sonnet)
Contrato ABERTO: Doc Reader extrai cargo → preenche campo profissão automaticamente no backend.

Contrato FECHADO: Doc Reader passa info para Julia (terminou em tal data, cargo X) → Julia pergunta ao lead se está trabalhando em outra profissão autônoma ou se está desempregado → após resposta do lead, Julia preenche profissão (não o subagente).
11
Fase 11 Lead completo — triagem Atendente / Julia
Com todos os dados e 3 documentos principais, lead está pronto para triagem e distribuição no CRM.

  • Disparo automático ao receber imagem/documento via WhatsApp
  • Análise com Claude Vision (atualmente haiku — MIGRAR para Sonnet)

  • Deduplicação por hash SHA256

Fluxo interno do subagente — 5 etapas
1
Validar
Legível? É o documento correto para a etapa? Se inválido → output para Julia com erro (ilegível ou documento errado).
2
Extrair
Dados com score de confiança por campo: ALTO / MÉDIO / BAIXO.
3
Preencher aba Dados
Campos ALTO: salvar imediatamente + badge verde. MÉDIO: badge amarelo. Conflito com dados existentes → alerta para Julia (não sobrescreve).
4
Salvar como PDF
Documento convertido e nomeado → aparece na aba Documentos do drawer com badge de confirmação.
5
Enviar JSON para Julia
Julia recebe contexto completo com todos os dados extraídos e scores para orquestrar os próximos passos da conversa.
Dados antes do cadastro — solução: Quando lead ainda não foi cadastrado (sem lead_id), o Doc Reader salva dados em whatsapp_conversas.metadata.pending_lead_data (JSONB). Quando atendente clica em "Cadastrar + Restrições", o botão lê esse JSONB e pré-preenche o formulário. Após cadastro criado, dados são migrados para leads e pending_lead_data é zerado.
