Seção 5 — Diagnóstico: Falha no Recebimento de Mídia

P0 — Impacto total no subagente de documentos
P0: Mídia inbound quebrada — documentos não chegam ao sistema
Todas as imagens e documentos recebidos via WhatsApp não estão sendo processados corretamente. O subagente de documentos depende inteiramente deste fluxo. As 4 causas abaixo estão ordenadas por probabilidade.
60% MAIS PROVÁVEL Causa 1 — n8n não está baixando arquivo da Meta
Quando WhatsApp envia imagem/documento, o webhook Meta não envia a URL do arquivo — envia apenas um media_id. O n8n precisa fazer 2 chamadas extras:
GET https://graph.facebook.com/v20.0/{media_id} Authorization: Bearer {WHATSAPP_TOKEN} → Retorna: { "url": "https://lookaside.fbsbx.com/..." } (expira ~5min!) GET {url_temporaria} Authorization: Bearer {WHATSAPP_TOKEN} → Retorna: binário do arquivo
Verificação: SELECT media_url FROM whatsapp_mensagens WHERE tipo IN ('image','document') AND direcao='entrada' LIMIT 10 — resultado está preenchido com URL do Storage ou null/media_id?
25% Causa 2 — Upload para Storage não está acontecendo
Mesmo baixando o binário, o n8n precisa fazer upload para o bucket correto no Supabase Storage. Fluxo correto completo:
Meta webhook → media_id → GET graph.facebook.com/{media_id} → URL temporária → GET URL temporária → binário → POST storage.supabase/{bucket}/{org_id}/{conversa_id}/{timestamp}-{filename} → UPDATE whatsapp_mensagens SET media_url = URL_DO_STORAGE
10% Causa 3 — Bucket privado + getPublicUrl() = HTTP 403
Se o bucket é privado, getPublicUrl() retorna URL que resulta em 403. Solução: usar createSignedUrl() com TTL ou tornar o bucket público.

Verificação: Tentar abrir a media_url de uma mensagem recebida no navegador. Dá 403? 404?
5% Causa 4 — n8n salvando URL temporária da Meta
A URL lookaside.fbsbx.com da Meta expira em ~5 minutos. Se o n8n salvar essa URL direto no media_url em vez de baixar e reuplodar → imagem carrega por 5min e quebra permanentemente.

Seção 6 — Spec Técnico: Preview e Recebimento de Mídia

Envio (Frontend → WhatsApp)
Ctrl+V: intercepta evento paste → extrai imagens do clipboard → blobUrl → MultiAttachmentPreview
Drag & Drop: onDrop → filtra tipos suportados → mesmo fluxo do Ctrl+V
Tipos aceitos:
image (jpg/png) 5MB
video (mp4/3gpp) 16MB
document (pdf/doc/xls) 100MB
audio 16MB
Fluxo: Upload → Storage bucket → INSERT otimista com status 'pending' → Edge Function → WhatsApp
Recebimento (WhatsApp → Frontend)
n8n: media_id → GET graph.facebook.com → URL temporária → baixar binário → upload Storage → salvar URL permanente
Segurança: Signed URLs se bucket privado (TTL configurável)
Renderização por tipo:
imageelemento <img> com fallback skeleton
videoelemento <video controls> com poster
audioelemento <audio controls> compacto
documentícone + nome do arquivo + tamanho formatado
