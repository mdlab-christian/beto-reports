Fluxo de Atendimento — 11 Fases

01 Lead entra via WhatsApp com CPF + interesse
02 Julia apresenta o processo e solicita documentos
03 Julia pede documento de identificação (CNH ou identidade)
04 Doc Reader processa identificação → extrai nome, CPF, nascimento, RG, sexo → salva PDF
Schema: nome, cpf, rg, data_nascimento, sexo, validade + confidence por campo
05 Julia pede comprovante de residência
06 Doc Reader processa comprovante
Nome bate: salva PDF + extrai endereço  |  Nome diverge: Julia pede declaração c/ foto
07 Sistema cadastra lead automaticamente com dados dos docs + WhatsApp
lead_coluna = 'em_atendimento'  |  pending_lead_data pré-preenche o formulário
08 Lead em 'em_atendimento' — aguardando profissão (CTPS)
09 Julia pede CTPS (carteira de trabalho)
10 Doc Reader processa CTPS → atualiza leads.profissao
Contrato aberto: último vínculo ativo  |  Contrato fechado: último vínculo registrado
11 Lead completo → pronto para triagem / distribuição no CRM

Backend — Migrations

M-01 whatsapp_templates — ADD deleted_at P0
ALTER TABLE public.whatsapp_templates ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ DEFAULT NULL; CREATE INDEX IF NOT EXISTS idx_whatsapp_templates_org_active ON public.whatsapp_templates(organizacao_id) WHERE deleted_at IS NULL;
M-02 leads_atividades — ADD updated_at + trigger P0
ALTER TABLE public.leads_atividades ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ DEFAULT now() NOT NULL; CREATE TRIGGER set_updated_at_leads_atividades BEFORE UPDATE ON leads_atividades FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
M-03 leads — TRIGGER sync awaiting_human / aguardando_humano P0
CREATE OR REPLACE FUNCTION sync_awaiting_human() RETURNS TRIGGER AS $$ BEGIN IF NEW.awaiting_human IS DISTINCT FROM OLD.awaiting_human THEN NEW.aguardando_humano := NEW.awaiting_human; ELSIF NEW.aguardando_humano IS DISTINCT FROM OLD.aguardando_humano THEN NEW.awaiting_human := NEW.aguardando_humano; END IF; RETURN NEW; END; $$ LANGUAGE plpgsql; CREATE TRIGGER sync_awaiting_human_trigger BEFORE UPDATE ON leads FOR EACH ROW EXECUTE FUNCTION sync_awaiting_human();
Campo duplicado: leads.awaiting_human e leads.aguardando_humano coexistem. Trigger mantém sincronismo até refactor futuro.
M-04 mdzap_icp_analises — CREATE TABLE (nova) P2
CREATE TABLE public.mdzap_icp_analises ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, organizacao_id UUID NOT NULL REFERENCES organizacoes(id) ON DELETE CASCADE, data_analise DATE NOT NULL, periodo TEXT CHECK (periodo IN ('daily', 'weekly')) NOT NULL, resultado JSONB NOT NULL DEFAULT '{}', created_at TIMESTAMPTZ DEFAULT now() NOT NULL, updated_at TIMESTAMPTZ DEFAULT now() NOT NULL, deleted_at TIMESTAMPTZ ); CREATE INDEX ON mdzap_icp_analises(organizacao_id, data_analise) WHERE deleted_at IS NULL; ALTER TABLE mdzap_icp_analises ENABLE ROW LEVEL SECURITY; CREATE POLICY "tenant_isolation" ON mdzap_icp_analises USING (organizacao_id = public.get_org_id()); CREATE TRIGGER set_updated_at BEFORE UPDATE ON mdzap_icp_analises FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
