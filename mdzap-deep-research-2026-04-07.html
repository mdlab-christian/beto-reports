P0 de Banco Identificados

E-01 · Crítico
Issue: whatsapp_templates sem deleted_at

Soft delete obrigatório em toda tabela. Tabela whatsapp_templates não possui a coluna deleted_at, violando regra P0 do MdFlow.

ALTER TABLE public.whatsapp_templates ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ DEFAULT NULL; CREATE INDEX IF NOT EXISTS idx_whatsapp_templates_org_active ON public.whatsapp_templates(organizacao_id) WHERE deleted_at IS NULL;
Issue: campo duplicado awaiting_human em leads

Tabela leads tem dois campos com a mesma semântica: awaiting_human e aguardando_humano. Necessário sincronizar via trigger até consolidação.

CREATE OR REPLACE FUNCTION sync_awaiting_human_to_aguardando() RETURNS TRIGGER AS $$ BEGIN NEW.aguardando_humano := NEW.awaiting_human; RETURN NEW; END; $$ LANGUAGE plpgsql; CREATE TRIGGER trg_sync_awaiting_human BEFORE UPDATE ON public.leads FOR EACH ROW EXECUTE FUNCTION sync_awaiting_human_to_aguardando();

Como usar o /projeto

O que é o /projeto

Sistema de gerenciamento de projetos complexos com banco de dados. Cada projeto tem etapas atômicas. CC executa 1 etapa por vez, registra resultado e avança para a próxima.

🎮
Manual
CC gera prompt, você cola. Mais controle.
⚡
Automático
CC executa sozinho em loop. Mais rápido.
Fluxo de Trabalho
Terminal
1. /projeto → iniciar projeto 2. /projeto → adicionar etapa [E-01: Fix deleted_at] 3. /projeto → adicionar etapa [E-02: Fix imagens] 4. ... (adicionar todas as etapas) 5. /projeto → executar projeto └— CC executa E-01, você testa, aprova 6. /projeto → próxima etapa └— CC executa E-02, você testa... 7. /projeto → html projeto 8. /projeto → finalizar projeto
Comandos Principais
iniciar projeto
Cria o projeto no banco, coleta nome / desc / páginas / modo de execução
adicionar etapa
Adiciona uma etapa (título, camada, PRD, critério de aceite)
executar projeto
Pega próxima etapa pendente, gera C-prompt, executa, valida resultado
html projeto
Gera HTML visual com progresso, PRDs e relatórios de cada etapa
listar projetos
Mostra todos os projetos ativos com progresso percentual
finalizar projeto
Marca como concluído após todas as etapas testadas = true
