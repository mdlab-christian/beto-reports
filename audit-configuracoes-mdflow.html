🚨 P0 — Bloqueadores (Corrigir antes do próximo deploy)

P0 Multi-tenant isolation: .update() e .delete() sem filtro organizacao_id
26 arquivos afetados

Queries executam atualização/exclusão apenas por ID, sem garantir que o registro pertence à organização do usuário. Isso permite que um usuário modifique/delete dados de outra organização (CRÍTICO).

// ❌ INSEGURO — não valida organizacao_id await supabase .from('docs_tipos') .update({ ativo: novoAtivo }) .eq('id', id); // Falta: .eq('organizacao_id', organizacao_id!) // ✅ CORRETO await supabase .from('docs_tipos') .update({ ativo: novoAtivo }) .eq('id', id) .eq('organizacao_id', organizacao_id!);

P0 Inline styles com backgroundColor dinâmico (sem fallback CSS)
TabProcessos.tsx:79, TabAtividades.tsx:203, TabEtiquetas.tsx:150

Cores armazenadas em banco de dados (hex) são aplicadas diretamente via style={{ backgroundColor: ... }}. Sem validação ou fallback, pode quebrar layout se cor for inválida.

// ❌ Problemático <span style={{ backgroundColor: sit.cor || 'hsl(var(--muted-foreground))' }} /> // ✅ Melhor — validar hex antes const isValidHex = /^#(?:[0-9a-f]{3}){1,2}$/i.test(cor); const bgColor = isValidHex ? cor : 'hsl(var(--muted-foreground))'; <span style={{ backgroundColor: bgColor }} />
