Etapa 2 — Primeiro Atendimento (Júlia)
Mensagem de boas-vindas com apresentação e proposta de valor
Apresentação: "Midas Restrições — especialistas em restrições indevidas de crédito"
Explica que a auditoria é gratuita
Pergunta se tem interesse em descobrir a situação do CPF
Se SIM: solicita RG/CNH + Comprovante de residência em nome do lead
Se NÃO: encerrar imediatamente. Zero follow-ups.

Etapa 3 — Subagente de Documentos: RG/CNH
Processamento automático do documento de identificação
Verifica legibilidade — se ilegível: Júlia pede foto melhor/PDF
Extrai: nome completo, data nascimento, CPF, RG
Popula dados automaticamente no sistema (lead)
🏠
Etapa 4 — Subagente de Documentos: Comprovante de Residência
Verificação de titularidade e extração de endereço
Verifica legibilidade
Verifica se está no nome do lead (nome OU CPF)
SE NÃO estiver no nome → Opção 1: banco digital (extrato/boleto com endereço)
SE ainda não tiver → Opção 2: declaração de terceiro
Modelo declaração: "Eu, [nome terceiro], portador CPF [X], declaro que [nome lead], portador CPF [Y], reside comigo" + doc identidade do terceiro
🔍
Etapa 5 — Auditoria (Humano)
Subagente passa output para Júlia → dados populados → humano executa
Dados populados: endereço, nome, CPF, RG, data nascimento
Júlia envia: "Vamos realizar a auditoria agora, em breve retorno com as informações"

HUMANO executa auditoria na Boa Vista (processo externo)

Seção 4 — Causa Raiz dos Problemas

S4
Problema 1 — Modais sem dados do lead no MdZap
🔴 Causa Raiz: lead_id NULL até criação manual — fluxo de criação automática não implementado 50%

🔴 Causa Raiz: prompts genéricos, fluxo Midas nunca foi configurado 80%
🟡 Alternativa A: Prompts existem mas não vinculados ao fluxo de mensagens inbound 15%
🔵 Alternativa B: Configuração de webhook inbound não aciona Júlia corretamente 5%
Problema 3 — Subagente de documentos inexistente
🔴 Causa Raiz: foi planejado mas nunca implementado — document-received na lista dos "11 workflows mortos" 90%
🟡 Alternativa A: Existe mas com nome diferente no n8n 7%
🔵 Alternativa B: Implementado como Edge Function Supabase, não como n8n 3%

P1
Criação automática do lead quando CPF chega
▼
📝 Trecho do Áudio
"Júlia já tem que cadastrar o lead automaticamente, no caso, o contato dele. E daí, conforme... conforme eu estava testando ali no MDZAP... vi que as coisas não mostrou porque não tinha lead vinculado à conversa"
🔍 Contexto Técnico
whatsapp_conversas.lead_id é nullable — lead_id null = modais em branco
Webhook inbound-upsert-lead existe e funciona (100%) mas não cria lead automaticamente
CPF pode vir junto com o nome na mensagem pré-preenchida pelo Facebook Ads
⚡ Opções
A
Júlia via n8n cria lead quando CPF chega (instante zero)
+ Simples, linear
- CPF pode ser inválido ou já existir
B
Sistema cria pré-lead com telefone quando primeira mensagem chega, enriquece com CPF quando confirmado RECOMENDADO
+ Robusto: captura lead mesmo sem CPF válido
- Requer upsert inteligente com deduplicação
C
Fluxo manual permanece, atendente cria lead após Doc Reader
+ Zero desenvolvimento
- Gargalo humano, modais sem dados até atendente agir
🔲 Resposta — O lead deve ser criado QUANDO?

Arquitetura do subagente de documentos
▼
📝 Trecho do Áudio
"o subagente de documentos vai extrair os dados... vai verificar... vai passar um output para a Júlia"
🔍 Contexto Técnico
Webhook document-received existe no n8n (nunca ativado)
Doc Reader atual é manual — usuário clica em botão no frontend

Arquivo relevante: /src/components/mdzap/lead-panel/doc-reader-helpers.ts
⚡ Opções
A
n8n workflow separado — ativado quando WhatsApp recebe imagem/PDF → Claude Vision → extrai → salva no lead → output para Júlia RECOMENDADO

- Requer webhook de recebimento de mídia no WhatsApp
B
Tool dentro do workflow da Júlia (Claude tool_use com visão)
+ Júlia tem contexto completo da conversa
- Acoplamento, Júlia fica mais lenta/complexa
C
Edge Function Supabase disparada por trigger de storage
+ Serverless, sem dependência do n8n
- Difícil de debugar, fora do padrão da arquitetura MdFlow
🔲 Resposta — Workflow n8n separado (A) ou tool dentro da Júlia (B)?
